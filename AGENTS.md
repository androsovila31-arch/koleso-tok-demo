# AGENTS.md

## Текущее состояние проекта

Сейчас в проекте есть простой статический одностраничный сайт без сборщика и без пакетного менеджера:

- `index.html` — структура страницы;
- `styles.css` — стили;
- `script.js` — мобильное меню.

## Стек проекта

- HTML
- CSS
- JavaScript без фреймворков и сборки

## Ключевые команды

Проект не требует обязательной сборки.

- Локальный просмотр: `node serve.js`, затем открыть `http://localhost:4173`
- Сборка: отсутствует
- Тесты: отсутствуют
- Линтинг/форматирование: отсутствуют

## Правила кода

- Не добавлять вымышленные команды, зависимости и инструменты, которых нет в проекте.
- Перед изменениями опираться на фактические файлы конфигурации и структуру репозитория.
- Новые технологии добавлять только вместе с явной конфигурацией проекта.
- Для каждого нового инструмента сразу фиксировать команду запуска, сборки и проверки.
- Поддерживать этот файл в актуальном состоянии при изменении стека.
- При разработке сложных фич или крупных рефакторингов использовать ExecPlan, как описано в `.agent/PLANS.md`, от этапа дизайна до реализации.

## Самопроверка перед завершением работы

- Убедиться, что все утверждения в документации соответствуют текущему состоянию репозитория.
- Проверить, появились ли в проекте файлы стека и конфигурации, которые требуют обновления этого файла.
- Если были добавлены команды, убедиться, что они действительно существуют в конфигурации проекта.
- Не писать, что тесты или сборка успешны, если такие команды ещё не настроены или не запускались.
